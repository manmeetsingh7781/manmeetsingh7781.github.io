The questions that always need to be asked before/after writing an algorithm is:

  1. How much time does this algorithm need to finish?
  1. How much space does this algorithm need for its computation?

To Standardize the Time/Space of algorithm is to use:

Big-O Notation

  • Gives an upper bound of the complexity in the worst case, helping to quantify performance as the input size becomes arbitrarily large.

  • Big O achieves consistency by focusing on the number of steps an Algorithm takes to complete a task.

  • Big O Notation generally refers to the worse-case scenario of an Algorithm.

  • Ignore Constants.

Question: If there are N data elements in an array, how many steps will linear search take?

Answer: worst-case scenario would be described as Big O(N)O(N)O(N)﻿

Question: if there are N data elements in an array, how many steps will reading from array take?

Answer: O(1)O(1)O(1)﻿ because reading from array is constant no matter what's the size of array is.

The Soul of Big O

  • How will an Algorithm's performance changes as the data increases?
  • Tells you the story of how the number of steps increase as the data changes
  • O(N)O(N)O(N)﻿'s performance is affected as we increase the data.

Deeper into the Soul of Big O

  • There is always an interceptioninterceptioninterception﻿ between two algorithms in which one performs better then the other from-and-to the certain point.

  • O(N)O(N)O(N)﻿ is not always a linear, if we need to search an element and its in the first cell of an array then we say it O(1)O(1)O(1)﻿ and its the best-case scenario. The worst-case would be O(N)O(N)O(N)﻿ when having go through each cell.

O(LogN)O (Log N)O(LogN)﻿ Explained

  • O(logN)O(log N)O(logN)﻿ means that for N data elements, the algorithm would take log2(N)log_2 (N)log2​(N)﻿ steps. If there are 888﻿ elements, the algorithm would take 3 steps. since log2(8)log_2(8)log2​(8)﻿ = 3.

  • Said another way, if we keep dividing 8 in half it would take us 3 steps until we end up with 1 element.

  • Said simply: O(logN)O(log N)O(logN)﻿ means the algorithm takes as many steps as it takes to keep halving the data elements until we remain with 1.

  • O(logN)O(log N)O(logN)﻿ takes just one additional steps each time data is doubled, while O(N)O(N)O(N)﻿ algorithm takes as many steps as there are data elements.
