Introduction

A self balancing BST is a tree that attempts to keep it's height, or the number of levels of nodes beneath the root, as small as possible at all times, automatically.

Question: What is the major disadvantage of an ordinary BST?

  • Most operations on a BST takes time proportional to the height of the tree, so it is desirable to keep height small.
  • Self-balancing binary trees solve this problem by performing transformations on the tree at key times (when inserting), in order to reduce the height.
  • Although a certain overhead is involved, it is justified in the long run by ensuring fast execution of later operations.
  • The height must always be at most the ceiling of log2(N) log_2(N)log2​(N)﻿.
  • BSTs are not always precisely balanced, since its expensive to keep a tree at minimum height at all times.

Delete Node - Algorithm

  1. Starting at root, find the deepest and rightmost node in binary tree and node which we want to delete.
  1. Replace the deepest rightmost node’s data with node to be deleted.
  1. Then delete the deepest rightmost node.

Applications of Self-Balancing BST

  1. They can be used to construct and maintain ordered list, such priority queues.
  1. No load factor
  1. They can be easily extended to record additional information or perform new operations. These extensions can be used in to optimize database queries or other list-processing algorithms.

Height

The height of tree is determined by the longest path from Root to the leaf node, do not count number of nodes, count the number of edges and return the Maximum from Left or Right side.

// code to find the maximum height of give node
protected int heightHelper(Node<T> root) {
    if (root != null) {
        int left = this.heightHelper(root.getLeft());
        int right = this.heightHelper(root.getRight());
        return Math.max(left, right) + 1;
    } else {
        return 0;
    }
}

AVL Tree - Balancing

  • Height balanced binary search tree, to balance a tree we calculate balance factor of each node.
  • Take the longest distance from root to subtree (left, right) and calculate the height.
  • Leaf nodes have balance factor of 0.

Balancing Properties

  • Balance-factor = height of left subtree - height of right subtree

Rotations

Rotations are done only on three nodes, by performing rotation to balance the tree

LL Rotation: Left and Left

LR Rotation: Left then Right

RR Rotation: Right and Right

RL Rotation: Right then Left
